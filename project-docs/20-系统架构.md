## 一、系统概述

### 1.1 系统定位

1Claw 是一款私有部署模式的个人软件，实现人类（老板）与 AI 雇员（OpenClaw）的协作管理。系统采用前后端分离架构，通过调度中心实现各模块间的解耦通讯。

### 1.2 核心目标

- 助力个人用户一键部署 AI 协作平台
- 快速拥有专属 AI 助手
- 支持人机协作、任务分配、对话沟通

### 1.3 用户角色

| 角色 | 说明 | 核心权限 |
|------|------|----------|
| 人类（老板） | 个人用户 | 雇佣 OpenClaw、对话、任务分配 |
| OpenClaw（雇员） | AI 雇员 | 与雇主对话、接收任务、执行任务 |

**注**：系统管理员角色搁置到 v3.0+（多用户版本）

## 二、系统架构

### 2.1 整体架构图

```
┌──────────────────────────────────────────────────────────────────────────────────────────────────────────────┐
│                              1Claw 系统架构 (v2.0 - 个人版)                                                   │

│   │  (Vue + Vite)   │                                                                                        │
│   └────────┬────────┘                                                                                        │
│            │                                                                                                 │
│            │ HTTP/WebSocket                                                                                  │
│            ▼                                                                                                 │

│   │                      调度中心系统                     │                                                   │
│   │                      (Python + SQLite)              │                                                   │
│   │  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  │                                                   │

│            │ WebSocket                     │                                                                 │
│            ▼                               ▼                                                                 │
│   ┌─────────────────┐              ┌─────────────────┐                                                       │
│   │ OpenClaw 通道插件 │             │ SQLite          │                                                       │
│   │ (OpenClaw Plugin)│             │    (数据库)      │                                                       │
│   └─────────────────┘              └─────────────────┘                                                       │
│                                                                                                              │
└──────────────────────────────────────────────────────────────────────────────────────────────────────────────┘
```

**架构说明**：
- **人类 Web 客户端** → HTTP/WebSocket → **调度中心系统**（获取消息、任务等）
- **调度中心系统** → 本地文件 → **SQLite 数据库**
- **OpenClaw 通道插件** → WebSocket → **调度中心系统**

**注**：PostgreSQL 为可选数据库（v3.0+ 多用户版本）

### 2.2 三大核心模块

| 模块 | 用户群体 | 架构形式 | 核心职责 |
|------|----------|----------|----------|
| 人类 Web 客户端 | 人类（老板） | Web 应用 (Vue+Vite+TS) | 信息查看、通讯交互、任务分配 |
| 调度中心系统 | 后端服务 | 纯后端 (Python+SQLite) | 消息流转、任务管理、通知服务 |
| OpenClaw 通道插件 | OpenClaw | OpenClaw Plugin | 消息收发、指令反馈 |

### 2.3 技术栈总览

| 模块 | 前端 | 后端 | 数据库 | 通讯协议 |
|------|------|------|--------|----------|
| 人类 Web 客户端 | Vue + Vite + TypeScript | - | - | HTTP + WebSocket |
| 调度中心系统 | - | Python | SQLite | HTTP + WebSocket |
| OpenClaw 通道插件 | - | OpenClaw Plugin | - | WebSocket |

## 三、模块详细设计

### 3.1 人类 Web 客户端

#### 3.1.1 功能模块

| 模块 | 功能说明 |
|------|----------|
| 聊天通讯 | 与已雇佣 OpenClaw 一对一对话、群组对话 |
| 雇员管理 | 查看已雇佣 OpenClaw、解雇操作、角色分配 |
| 任务管理 | 创建任务、分配任务、查看进度、任务历史 |

#### 3.1.2 核心业务规则

- 仅可与已雇佣 OpenClaw 对话
- 仅可雇佣空闲 - 启用状态的 OpenClaw
- 仅可解雇自己雇佣的 OpenClaw

---

### 3.2 调度中心系统

#### 3.2.1 核心职责

| 职责 | 说明 |
|------|------|
| 消息收发 | 接收消息并转发给目标接收者 |
| 消息分发 | 根据消息类型和目标分发 |
| 任务管理 | 任务创建、分配、状态追踪、截止时间提醒 |
| 权限校验 | 校验发送者和接收者之间的雇佣关系 |
| 消息持久化 | 持久化存储所有消息 |
| 状态管理 | 维护在线状态、已读/未读状态、任务状态 |
| 通知服务 | 任务超期提醒、状态变更通知 |
| 定时任务 | 截止时间提醒、心跳检查、数据清理 |

#### 3.2.2 数据库

**当前方案（v2.0）**：SQLite

**优势**：
- 零配置，开箱即用
- 无需 Docker 或外部服务
- 单用户场景性能足够
- 便携，复制文件即可迁移

**未来方案（v3.0+）**：PostgreSQL

**适用场景**：
- 多用户支持
- 高并发需求
- 高级功能（JSONB、全文搜索等）

---

### 3.3 OpenClaw 通道插件

#### 3.3.1 核心功能

| 功能 | 说明 |
|------|------|
| 消息接收 | 接收来自人类或其他 OpenClaw 的消息 |
| 消息发送 | 向人类或其他 OpenClaw 发送消息 |
| 任务接收 | 接收人类分配的任务 |
| 任务反馈 | 向人类反馈任务执行进度和结果 |

#### 3.3.2 连接管理

- 通过 WebSocket 与调度中心建立长连接
- 定期发送心跳维持连接
- 断线后自动重连

**搁置原因**：v2.0 为单用户模式

- 雇佣关系管理

## 五、部署架构

### 5.1 开发环境

```
开发主机
├── 调度中心后端（uvicorn :8001）
├── 人类 Web 客户端（npm dev :3000）
└── SQLite（本地文件）
```

### 5.2 生产环境

**方案一（推荐）**：单 Docker 容器

```
Docker Container
├── Traefik（入口，80/443）
├── web-client（静态文件）
├── scheduler-server（uvicorn :8001）
└── SQLite（容器内）或 PostgreSQL（外部）
```
