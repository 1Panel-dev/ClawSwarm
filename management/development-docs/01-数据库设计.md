## 二、表结构详细设计

### 2.1 humans (人类账号表)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PRIMARY KEY | 自增主键 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| email | VARCHAR(100) | UNIQUE, NOT NULL | 邮箱 |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt 加密的密码 |
| status | VARCHAR(20) | DEFAULT 'enabled' | 状态：enabled/disabled/deleted |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 更新时间 |
| deleted_at | TIMESTAMP | NULL | 删除时间（软删除） |

**索引**:
- `idx_humans_status`: status 字段索引
- `idx_humans_created_at`: created_at 字段索引

---

### 2.2 openclaws (OpenClaw 表)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PRIMARY KEY | 自增主键 |
| name | VARCHAR(100) | UNIQUE, NOT NULL | 名称 |
| description | TEXT | NULL | 描述 |
| skills | TEXT[] | NULL | 技能标签数组 |
| status | VARCHAR(20) | DEFAULT 'idle' | 状态：idle/hired/disabled |
| can_message_peers | BOOLEAN | DEFAULT TRUE | 是否可给其他 OpenClaw 发消息 |
| workspace_path | VARCHAR(255) | NULL | 工作空间路径 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 更新时间 |
| disabled_at | TIMESTAMP | NULL | 禁用时间 |

**索引**:
- `idx_openclaws_status`: status 字段索引
- `idx_openclaws_created_at`: created_at 字段索引

---

### 2.3 employments (雇佣关系表)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PRIMARY KEY | 自增主键 |
| human_id | INTEGER | FK → humans.id, NOT NULL | 人类 ID |
| openclaw_id | INTEGER | FK → openclaws.id, NOT NULL | OpenClaw ID |
| status | VARCHAR(20) | DEFAULT 'active' | 状态：active/terminated |
| hired_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 雇佣时间 |
| terminated_at | TIMESTAMP | NULL | 解雇时间 |
| terminated_reason | VARCHAR(255) | NULL | 解雇原因：manual/auto |

**索引**:
- `idx_employments_human_id`: human_id 索引
- `idx_employments_openclaw_id`: openclaw_id 索引
- `idx_employments_status`: status 字段索引
- `idx_employments_hired_at`: hired_at 字段索引
- `unique_active_employment`: UNIQUE(human_id, openclaw_id, status) - 防止重复雇佣

**外键约束**:
- `fk_employments_human`: human_id REFERENCES humans(id) ON DELETE CASCADE
- `fk_employments_openclaw`: openclaw_id REFERENCES openclaws(id) ON DELETE CASCADE

---

### 2.4 system_logs (系统日志表)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PRIMARY KEY | 自增主键 |
| operation_type | VARCHAR(50) | NOT NULL | 操作类型 |
| operator | VARCHAR(100) | NOT NULL | 操作者 |
| target_type | VARCHAR(50) | NULL | 目标类型 |
| target_id | INTEGER | NULL | 目标 ID |
| details | JSONB | NULL | 操作详情（JSON 格式） |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**索引**:
- `idx_logs_operation_type`: operation_type 索引
- `idx_logs_operator`: operator 索引
- `idx_logs_target`: target_type + target_id 复合索引
- `idx_logs_created_at`: created_at 索引（用于定期清理）

**操作类型枚举**:
```
human_create, human_update, human_delete
human_enable, human_disable
openclaw_create, openclaw_update
openclaw_enable, openclaw_disable, openclaw_peer_message
employment_hire, employment_terminate
role_create, role_update, role_delete
memory_clear, memory_export
setting_update
```

---

### 2.5 roles (角色模板表)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PRIMARY KEY | 自增主键 |
| name | VARCHAR(100) | UNIQUE, NOT NULL | 角色名称 |
| description | TEXT | NULL | 描述 |
| role_type | VARCHAR(20) | DEFAULT 'preset' | 类型：preset/custom |
| agent_config | JSONB | NULL | Agent 配置 |
| skill_set | JSONB | NULL | Skill 集合 |
| work_knowledge | JSONB | NULL | 工作知识 |
| usage_count | INTEGER | DEFAULT 0 | 被使用次数 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

**索引**:
- `idx_roles_role_type`: role_type 索引
- `idx_roles_usage_count`: usage_count 索引

---

### 2.6 memories (记忆数据表)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PRIMARY KEY | 自增主键 |
| openclaw_id | INTEGER | FK → openclaws.id, NOT NULL | OpenClaw ID |
| human_id | INTEGER | FK → humans.id, NULL | 所属人类 ID（共享记忆时有值） |
| memory_type | VARCHAR(20) | DEFAULT 'independent' | 类型：independent/shared |
| content | JSONB | NULL | 记忆内容 |
| size_bytes | INTEGER | DEFAULT 0 | 大小（字节） |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

**索引**:
- `idx_memories_openclaw_id`: openclaw_id 索引
- `idx_memories_human_id`: human_id 索引
- `idx_memories_type`: memory_type 索引

**外键约束**:
- `fk_memories_openclaw`: openclaw_id REFERENCES openclaws(id) ON DELETE CASCADE
- `fk_memories_human`: human_id REFERENCES humans(id) ON DELETE SET NULL

---

### 2.7 system_settings (系统配置表)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PRIMARY KEY | 自增主键 |
| key | VARCHAR(100) | UNIQUE, NOT NULL | 配置键 |
| value | JSONB | NOT NULL | 配置值（JSON 格式） |
| description | TEXT | NULL | 描述 |
| updated_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

**预置配置**:
```json
{
  "employment_limit_per_human": {
    "value": 3,
    "description": "每个人类账号可雇佣的 OpenClaw 数量上限"
  },
  "log_retention_days": {
    "value": 60,
    "description": "系统日志保留天数（2 个月）"
  },
  "openclaw_workspace_quota_mb": {
    "value": 1024,
    "description": "每个 OpenClaw 工作空间配额（MB）"
  }
}
```

## 五、数据字典

### 5.1 状态枚举

#### humans.status
| 值 | 说明 |
|----|------|
| enabled | 启用（可登录、可雇佣） |
| disabled | 禁用（不可登录、OpenClaw 自动解雇） |
| deleted | 已删除（软删除） |

#### openclaws.status
| 值 | 说明 |
|----|------|
| idle | 空闲（可被雇佣） |
| hired | 已雇佣 |
| disabled | 已禁用 |

#### employments.status
| 值 | 说明 |
|----|------|
| active | 雇佣中 |
| terminated | 已解雇 |

#### employments.terminated_reason
| 值 | 说明 |
|----|------|
| manual | 手动解雇 |
| auto | 自动解雇（人类账号禁用触发） |

#### memories.memory_type
| 值 | 说明 |
|----|------|
| independent | 独立记忆 |
| shared | 共享记忆 |

#### roles.role_type
| 值 | 说明 |
|----|------|
| preset | 预制角色（管理员创建） |
| custom | 自定义角色 |

## 六、日志清理策略

系统日志保留 **60 天（2 个月）**，定期清理：

```sql
-- 清理 60 天前的日志
DELETE FROM system_logs 
WHERE created_at < NOW() - INTERVAL '60 days';
```

**建议**: 使用 cron 任务每天执行一次清理。
