## 一、部署方式

根据开发框架规范，管理系统采用以下部署方式：

| 组件 | 部署方式 | 说明 |
|------|----------|------|
| 后端 | 直接部署 | Python FastAPI 应用 |
| 前端 | Nginx 静态托管 | Vite 构建后的静态文件 |
| 数据库 | PostgreSQL | 独立服务 |

## 四、数据库部署

### 4.1 安装 PostgreSQL

```bash
# Ubuntu
sudo apt update
sudo apt install postgresql postgresql-contrib

# CentOS
sudo yum install postgresql-server postgresql-contrib
```

### 4.2 创建数据库和用户

```bash
sudo -u postgres psql

-- 创建数据库
CREATE DATABASE management_db;

-- 创建用户
CREATE USER management_user WITH PASSWORD 'your_secure_password';

-- 授权
GRANT ALL PRIVILEGES ON DATABASE management_db TO management_user;
\q
```

### 4.3 初始化 Schema

```bash
# 运行 Alembic 迁移
cd /home/node/.openclaw/github-workspace/1claw/management-backend
source venv/bin/activate
alembic upgrade head
```

## 五、后端部署

### 5.1 准备环境

```bash
# 创建部署目录
mkdir -p /home/node/.openclaw/github-workspace/1claw/management-deploy/backend
cd /home/node/.openclaw/github-workspace/1claw/management-deploy/backend

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install --upgrade pip
pip install -r /home/node/.openclaw/github-workspace/1claw/management-backend/requirements.txt
```

### 5.2 配置文件

创建 `.env` 文件：

```bash
# /home/node/.openclaw/github-workspace/1claw/management-deploy/backend/.env

# 数据库
DATABASE_URL=postgresql://management_user:your_secure_password@localhost:5432/management_db

# JWT 配置
JWT_SECRET_KEY=your_jwt_secret_key_here
JWT_ALGORITHM=HS256
JWT_EXPIRATION_MINUTES=1440

# 服务器
HOST=0.0.0.0
PORT=8000
WORKERS=4

# 日志
LOG_LEVEL=INFO
LOG_FILE=/home/node/.openclaw/github-workspace/1claw/management-deploy/logs/backend.log

# 工作空间根目录
WORKSPACE_ROOT=/home/node/.openclaw/github-workspace/1claw/workspaces
```

### 5.3 启动后端

**开发环境**:
```bash
cd /home/node/.openclaw/github-workspace/1claw/management-deploy/backend
source venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

**生产环境** (使用 systemd):

创建 systemd 服务文件：

```ini
# /etc/systemd/system/management-backend.service
[Unit]
Description=1Claw Management Backend
After=network.target postgresql.service

[Service]
Type=notify
User=node
WorkingDirectory=/home/node/.openclaw/github-workspace/1claw/management-deploy/backend
Environment="PATH=/home/node/.openclaw/github-workspace/1claw/management-deploy/backend/venv/bin"
ExecStart=/home/node/.openclaw/github-workspace/1claw/management-deploy/backend/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable management-backend
sudo systemctl start management-backend
sudo systemctl status management-backend
```

## 六、前端部署

### 6.1 构建前端

```bash
cd /home/node/.openclaw/github-workspace/1claw/management-frontend

# 安装依赖
npm install

# 生产环境构建
npm run build
```

构建产物在 `dist/` 目录。

### 6.2 配置 Nginx

```nginx
# /etc/nginx/sites-available/management-frontend
server {
    listen 80;
    server_name management.1claw.local;

    root /home/node/.openclaw/github-workspace/1claw/management-deploy/frontend/dist;
    index index.html;

    # 静态文件
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # 缓存静态资源
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

启用站点：

```bash
sudo ln -s /etc/nginx/sites-available/management-frontend /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 七、日志清理

### 7.1 系统日志数据库清理

创建 cron 任务：

```bash
# /etc/cron.daily/cleanup-management-logs
#!/bin/bash
psql -U management_user -d management_db -c "DELETE FROM system_logs WHERE created_at < NOW() - INTERVAL '60 days';"
```

```bash
sudo chmod +x /etc/cron.daily/cleanup-management-logs
```

### 7.2 应用日志轮转

```bash
# /etc/logrotate.d/management-backend
/home/node/.openclaw/github-workspace/1claw/management-deploy/logs/*.log {
    daily
    rotate 30
    compress
    delaycompress
    missingok
    notifempty
    create 0644 node node
}
```

## 八、健康检查

### 8.1 后端健康检查接口

```
GET /api/admin/health
```

**响应**:
```json
{
  "status": "healthy",
  "database": "connected",
  "timestamp": "2026-03-17T15:00:00Z"
}
```

### 8.2 监控脚本

```bash
#!/bin/bash
# /home/node/.openclaw/github-workspace/1claw/management-deploy/healthcheck.sh

# 检查后端服务
if ! systemctl is-active --quiet management-backend; then
    echo "Backend service is down!"
    exit 1
fi

# 检查数据库连接
if ! psql -U management_user -d management_db -c "SELECT 1" > /dev/null 2>&1; then
    echo "Database connection failed!"
    exit 1
fi

# 检查 Nginx
if ! systemctl is-active --quiet nginx; then
    echo "Nginx is down!"
    exit 1
fi

echo "All services healthy"
exit 0
```

## 九、备份策略

### 9.1 数据库备份

```bash
#!/bin/bash
# /home/node/.openclaw/github-workspace/1claw/management-deploy/backup-db.sh

BACKUP_DIR="/home/node/.openclaw/github-workspace/1claw/management-deploy/backups"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
pg_dump -U management_user management_db > $BACKUP_DIR/management_db_$DATE.sql

# 压缩
gzip $BACKUP_DIR/management_db_$DATE.sql

# 清理 30 天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +30 -delete

echo "Backup completed: management_db_$DATE.sql.gz"
```

### 9.2 备份计划

```bash
# /etc/cron.daily/backup-management-db
#!/bin/bash
/home/node/.openclaw/github-workspace/1claw/management-deploy/backup-db.sh
```

## 十一、常见问题

### 11.1 后端无法启动

```bash
# 检查日志
sudo journalctl -u management-backend -f

# 检查端口占用
sudo lsof -i :8000

# 检查数据库连接
psql -U management_user -d management_db -c "SELECT 1"
```

### 11.2 前端页面无法访问

```bash
# 检查 Nginx 状态
sudo systemctl status nginx

# 检查 Nginx 配置
sudo nginx -t

# 检查 Nginx 日志
sudo tail -f /var/log/nginx/error.log
```

### 11.3 数据库连接失败

```bash
# 检查 PostgreSQL 状态
sudo systemctl status postgresql

# 检查用户权限
sudo -u postgres psql -c "\du"
```
